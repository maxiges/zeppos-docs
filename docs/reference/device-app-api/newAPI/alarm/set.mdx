# set

:::note
Start from API_LEVEL `3.0` . Please refer to [API_LEVEL](../../../../guides/framework/device/compatibility.md).
:::

Support for persistent timers to wake up pages of Mini Program.

:::info
permission code: `device:os.alarm`
:::

## Type

```ts
function set(option: Option): Result
```

## Parameters

### Option

| Property | Type | Required | DefaultValue | Description | API_LEVEL |
| :--- | :--- | :--- | :--- | :--- | :--- |
| appid | `number` | N | - | App ID of the Mini Program, default current Mini Program ID. | 3.0 |
| url | `string` | Y | - | File path to wake up Mini Program, supporting App Service. | 3.0 |
| time | `number` | N | - | Timer execution time (UTC timestamp, in seconds). Higher priority than `delay`. | 3.0 |
| delay | `number` | N | - | Delay in seconds from current time. Required if `time` is not set. | 3.0 |
| param | `string` | N | - | Argument passed to `app.js` lifecycle `onCreate`. | 3.0 |
| store | `boolean` | N | `false` | If `true`, the timer persists after device reboot. | 3.0 |
| repeat_type | `number` | N | `REPEAT_ONCE` | Repetition type (e.g., `REPEAT_MINUTE`, `REPEAT_HOUR`, `REPEAT_DAY`, `REPEAT_WEEK`). | 3.0 |
| repeat_period | `number` | N | `1` | Interval value for the selected `repeat_type`. E.g., if `repeat_type` is `REPEAT_MINUTE` and `repeat_period` is 10, it repeats every 10 minutes. | 3.0 |
| repeat_duration | `number` | N | `1` | Defines how many occurrences are allowed within the defined period. Usually set to 1. | 3.0 |
| week_days | `number` | N | - | Used with `REPEAT_WEEK`. Bitwise mask of week constants (e.g., `WEEK_MON \| WEEK_WED`). | 3.0 |
| start_time | `number` | N | - | UTC timestamp when the repeating reminder starts. | 3.0 |
| end_time | `number` | N | - | UTC timestamp when the repeating reminder ends. | 3.0 |

### Result

| Type                | Description                                                                                                                                                                                  |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <code>number</code> | The id returned by the timer creation, `0` is an invalid ID, which means the timer creation failed, and the ID remains the same after the system restart for timers that support persistence |

## Constants

### Timer repeats constants

| Constant        | Description                             | API_LEVEL |
| --------------- | --------------------------------------- | --------- |
| `REPEAT_ONCE`   | Repeat once                             | 3.0       |
| `REPEAT_MINUTE` | Specify the repetition period as minute | 3.0       |
| `REPEAT_HOUR`   | Specify the repetition period as hour   | 3.0       |
| `REPEAT_DAY`    | Specify the repetition period as day    | 3.0       |
| `REPEAT_WEEK`   | Specify the repetition period as week   | 3.0       |
| `REPEAT_MONTH`  | Specify the repetition period as month  | 3.0       |
| `REPEAT_YEAR`   | Specify the repetition period as year   | 3.0       |

### Timer weekly constants

| Constant   | Description | API_LEVEL |
| ---------- | ----------- | --------- |
| `WEEK_MON` | Monday      | 3.0       |
| `WEEK_TUE` | Tuesday     | 3.0       |
| `WEEK_WED` | Wednesday   | 3.0       |
| `WEEK_THU` | Thursday    | 3.0       |
| `WEEK_FRI` | Friday      | 3.0       |
| `WEEK_SAT` | Saturday    | 3.0       |
| `WEEK_SUN` | Sunday      | 3.0       |

// Once daily at 05:30
// Calculate delay until 05:30 next time
const now = new Date()
let target = new Date(now.getFullYear(), now.getMonth(), now.getDate(), 5, 30, 0)
if (target < now) {
  target.setDate(target.getDate() + 1)
}
const delay = Math.floor((target.getTime() - now.getTime()) / 1000)

set({
  url: 'pages/index.js',
  repeat_type: REPEAT_DAY,
  repeat_period: 1,
  repeat_duration: 1,
  delay: delay,
  store: true
})
